Prework Study Guide

HTML

  • The head element contains information (metadata) about the webpage
  • The body element represents the visible content shown to the user
  • The title element defines the webpage and is very important for Search Engine Optimization (SEO)
  • Style elements define CSS associated with the website
  • Base elements define the base URL for a webpage
  • Link elements connect an external resource to the HTML document
  • Meta elements define metadata such as character set, description, keywords, author and viewport
  • Script elements embed client-side scripts in the HTML document
  • h1 - h6 elements represent the level of heading in text block represents
  • P element represents a paragraph
  • ul= unordered list, ol= ordered list, li= list
  • br element creates an empty space (line break) between two blocks of content

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Assigning colors can be done by using semantic references such as red or blue; Or refer to a color by its hexadecimal code or hex #
  • Any HTML element assigned to the class ".card" will gain the properties from the css

Git

  • git status: checks what branch we are currently on
  • git checkout - b branch-name: creates a new branch and switches to it
  • git merge: merge remote branch to current branch to update it
  • git pull: fetch and merge commits from the tracking remote branch
  • git clone: retrieve repository from a hosted location via URL
  • git log: show all commits in the current branch's history
Steps to save work to GitHub
  • Stage the changes by using "git add -A" in the terminal inside prework-study-guide
  • Commit the changes by using 'git commit -m "added code to HTML file"
  • Use the following code to pull in the base branch before pushing any code "git pull origin main"
  • Push all of the changes by using the command "git push origin feature/add-html"
  • Navigate to prework-study-guide in repo on GitHub, once there navigate to feature/add-html branch
  • Select "pull Requests", then "New pull request"
  • Give the pull request a title and short description
  • Click "Create pull request"
  • Click "Merge pull request"
  • Update in terminal by using "git checkout main" "git pull origin main"
